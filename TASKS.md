**What:** Remove or replace all `any` types and fix unused variable/import warnings so `npm run lint` passes.

**Where:**

## 2. Fix React: escape apostrophe in API-dashboard.tsx

**What:** Fix the `react/no-unescaped-entities` violation: an unescaped `'` (apostrophe) in JSX around line 325 in `src/components/API-dashboard.tsx`.

**Why:** React/ESLint warns on raw `'` in text to avoid broken markup and encoding issues. Use `&apos;`, `{"'"}` or move the text to a variable.

## 4. Add .env.example

**What:** Add a `.env.example` in the project root listing all required environment variable names with placeholder or dummy values (no real secrets). Include:
- `NEXT_PUBLIC_GOOGLE_CLIENT_ID`
- `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`
- `RESEND_API_KEY`
- `NEXT_PUBLIC_APP_URL`
- `ADMIN_SECRET` (for API key generation)
- `SCHEDULER_API_KEY` (for `/api/public/*` proxy)

**Why:** New developers and deployments need a single reference for required env vars. `.env.local` is gitignored, so `.env.example` is the standard way to document configuration without committing secrets.
